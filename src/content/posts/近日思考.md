## 左手腕禁止扭动
周一体测，因为 50 米测试中喊口令的同学声音太小，导致边上的人没有听到，已经快要跑到终点的我只能折返回来重新开始，但是刚跑到起点还没稳住身子，“跑”的口令又不合时宜地来了——起步慢了。不甘心的我跟着最后一组跑第三遍，终于在第三遍的途中，脚跟不上腿，腿跟不上上半身，整个人像是被扔出去的标枪，狠狠地扎在了跑道上。

幸运的是，虽然摔得很惨，但还好是侧面着地，手也支撑到位。不幸的是，左胳膊肘被擦破，手腕被扭到，后面的 1000 米测试也只能等到下周一补测了，讨厌这种开局。

## 新的设备——新的习惯
![周日下午拍摄于逸夫楼八楼自习室](https://img.theslowmo.com/2026/04/4ab3a3e0b4b4ecd266ef4d22fab5b8b2.webp "width=78%")

是的，我终于购入了心心念念的 MacBook Air。苦拯救者续航久矣，我始终想入手一台便携、续航时间长、性能又不差的设备，所以入手了图中 32GB 内存、512GB 存储的 MacBook Air M5 13 英寸版本，搭配镜像或扩展屏幕的 iPad，形成了一个新的工作站。

重型项目的开发和编译工作依旧是宿舍的拯救者负责，通过 MacBook 也可以方便地远程控制开发，而轻型的一些工作，慢慢地转向了图中的搭配，比如现在正在写的博客项目已经在 Mac 上同步部署了，日常的写作和一些轻量级的开发也都在 Mac 上进行，在自习室的网课学习也完全胜任，优秀的设备生态，十几个小时超强的续航时间，足以支持我在自习室待上一整天。

谈到远程开发，我尝试了多种远程控制的方式，比如 Parsec、ToDesk、Tailscale 虚拟局域网搭配 Windows Remote Desktop，效果都不尽人意，主要原因就是因为没有公网 IP 导致网络延迟高、画面清晰度低、丢包率高甚至掉线的问题。最终我选择了目前免费使用的一站式远程控制工具——UU 远程，非常简单易上手，只需要在设备上安装客户端并登录同一账号，即可远程控制。也就是这个工具，让我在逸夫楼八楼的自习室里用着免费的无线网也能流畅控制宿舍里的拯救者了，这种感觉确实让人欲罢不能。

而镜像扩展功能则是得益于苹果强大的生态系统提供的功能——"随航"。借助这个功能，我可以将 iPad 作为 Mac 的第二块屏幕使用，效果非常好。因为是原生支持的功能，所以丝毫感觉不到延迟和卡顿，另外在无线之外还能通过有线连接达到同时共享又充电的效果。

我逐渐习惯了这样的生活，我甚至在这种过程中感到时间仿佛变得更快了。

## 即将到来的软工课设实习
面对即将到来的软件工程课设实习，我不知道该说什么——实习内容与教学计划完全脱节。这学期才刚开数据库必修，人机交互（前端内容）还是选修课，上学期也只学过 C 语言和 Java 基础。这样的基础，如何支撑我们在几周内完成一个完整的 B/S 或 C/S 架构项目？

我不清楚这个实习内容是面向大多数学生还是实验室里的学生，如果是面对大多数的学生来讲未免要求过高，因为绝大多数的学生都没有前端开发，后端开发的经验，数据库更甚。我也只有在上学期系统学习过一段时间的全栈开发流程，不至于一头雾水。但是其他不是实验室里的同学，没有自学能力的学生怎么办，我想这很难说。

幸运的是，这是一个小组项目，但是完全零基础的同学不在少数，根本无法分配相应的工作，只能是有经验的同学承担绝大多数的工作。

总之，感觉这个实习内容和教学计划完全脱节了，甚至有点不负责任的感觉。我想会有很多组会有以下选择：

1. 选择一个非常简单的项目，能在几周内完成的那种，来保证能完成实习要求。
2. 选择一个稍微复杂一点的项目，但是几乎全部交给有能力的同学。
3. 借助 Agent，只需要购买 Coding Plan 即可通过 VibeCoding 完成项目。
4. 上咸鱼购买现成项目，或者使用学长的祖传项目。
5. 组员系统学习一下前端开发、后端开发、数据库，各自分工完成项目。

我认为最好的办法莫过于第三种，第一种很难达成项目要求，第二种又实在不太公平，第四种过于危险，第五种又不太现实，毕竟时间有限，学习时间成本太高，很难达到实际要求。于是我选择了第三种，借助 Claude Code、Codex 完成困难的项目开发工作，达到在实际的项目中学习的效果，而不是一头雾水地扑进不切实际的现场从零开始中去。

也许你会觉得我这种方法是投机取巧的、不负责任的，但我始终认为，实习的目的不在于完成一个项目，而在于是否能通过这个项目来学习一些东西。如果这个项目的要求过高，超出了大多数学生的能力范围，那么这个实习就失去了意义。借助 Agent 来完成项目开发工作，可以让我们更好地学习如何使用这些工具来提高开发效率，同时也能保证完成实习要求，这才是最重要的。

脱离教学计划的实习内容，往往会被解释为把有自学能力的学生标榜为上进，我想这本身就是不负责任的。这种实习只会让有技术的学生更有优势，而现有技术基础稍差的学生仿佛直接就会被抛弃。

这确实很残忍，但我同时也认为，这也是一种现实，毕竟在实际的工作中，技术能力是非常重要的。现有能力弱的学生抑或绝大多数的学生不得不做出选择，是选择花足够的时间与精力真正地系统学习积累经验，还是得过且过地完成实习要求，我想这将会很重要。
